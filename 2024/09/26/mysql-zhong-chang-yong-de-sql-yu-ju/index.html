MySQL中常用的SQL语句

MySQL SQL
MySQL
发布日期:   2024-09-26

记录MySQL数据库中常用的SQL语句，包括增删改查、过程函数。下面所用的数据和例子均在数据库 test_sample 中进行操作。所用到的数据如下：

  • 数据库名： test_sample
  • 表名： user_info

1. 表操作

A. 建表

创建一张表 user_info ，创建表的SQL语句如下：

CREATE TABLE `user_info`  (
  `id` int(255) NOT NULL AUTO_INCREMENT,
  `user_id` varchar(11) CHARACTER SET utf8 COLLATE utf8_general_ci NULL DEFAULT NULL,
  `name` varchar(80) CHARACTER SET utf8 COLLATE utf8_general_ci NULL DEFAULT NULL,
  `age` int(11) NULL DEFAULT NULL,
  `create_time` varchar(40) CHARACTER SET utf8 COLLATE utf8_general_ci NULL DEFAULT NULL,
  PRIMARY KEY (`id`) USING BTREE
) ENGINE = InnoDB AUTO_INCREMENT = 11 CHARACTER SET = utf8 COLLATE = utf8_general_ci ROW_FORMAT = Compact;

该语句为Navicat转储的SQL文件中的语句：

  • id：int 类型，长度 255，不能为空，自动递增。
  • user_id ：varchar 类型，长度 11，字符集设置为utf8， 可以为空， 默认值为空。
  • name ：varchar 类型，长度 80，字符集设置为utf8， 可以为空， 默认值为空。
  • age ：int 类型，长度 11，可以为空， 默认值为空。
  • create_time ：varchar 类型，长度 40，字符集设置为utf8， 可以为空， 默认值为空。
  • 主键为 id 。
ENGINE = InnoDB AUTO_INCREMENT = 11 CHARACTER SET = utf8 COLLATE = utf8_general_ci ROW_FORMAT = Compact;

这句话是对数据库中的字符集设置，字符使用utf8编码，并且默认的引擎是InnoDB。

B. 删表

删除表的SQL语句如下：

DROP TABLE IF EXISTS `user_info`;

当表存在的时候才删除表。

2. 增删改查 of 记录

记录的增删改查是针对表中的一条条数据的，也就是表中的记录。可以从表中插入/删除/修改/查询若干条记录。

A. 增加 insert

从表中插入若干条记录，语法如下：

INSERT INTO <表名>(若干个字段) VALUES (字段对应的值);

例子：

INSERT INTO test_sample.user_info (id, user_id, name, age, create_time`) VALUES (1, '41000000001', 'zhangsan', 3, '2024-07-30 16:58:51');

B. 删除 delete

从表中删除若干条记录，语法如下：

DELETE FROM <表名> WHERE <筛选条件>;

例子：

DELETE FROM test_sample.user_info WHERE id = 1;

C. 修改 update

从表中修改若干条记录，语法如下：

UPDATE <表名> SET <字段名> = <新值> WHERE <筛选条件>;

例子：

UPDATE test_sample.user_info SET age = 18 WHERE id = 1;

D. 查询 select

从表中查询若干条记录，语法如下：

SELECT <字段名> FROM <表名> WHERE <筛选条件>;

例子：

SELECT * FROM user_info;

3. 过程 Procedure

MySQL 5.0版本开始支持存储过程（Stored Procedure），是一种在数据库中存储复杂程序，以便外部程序调用的一种数据对象。存储过程是为了完成特定功能的SQL语句集，经编译创建并保存在数据库中，用户可通过制定存储过程的名字并给定参数来调用执行。

过程可以用来批量造数据。假如，现在想要在数据库中插入10000条数据，以进行测试。那么，就需要循环执行插入语句，让程序跑起来。循环执行SQL语句，直接对数据库进行操作，显然更高效快捷，和用代码去实现循环（还需要建项目、连接数据库等）。

A. 例子

以插入10条数据为例，MySQL数据库中运行的SQL语句示例如下：

CREATE DEFINER=`root`@`localhost` PROCEDURE `insertUserData`()
BEGIN
	DECLARE i INT DEFAULT 1;
	DECLARE name VARCHAR(40) DEFAULT 'zhangsan';
	DECLARE age INT DEFAULT 3;
	DECLARE create_time VARCHAR(40) DEFAULT '2024-07-30 16:58:51';

	start TRANSACTION;
	WHILE i <= 10 DO
		INSERT INTO user_info(`id`, `user_id`, `name`, `age`, `create_time`) VALUES (i, CONCAT(410, LPAD(i, 8, '0')), name, age, create_time);
		SET i = i + 1;
	END WHILE;
	commit;
END

B. 解析

1. 建函数

CREATE DEFINER=`root`@`localhost` PROCEDURE `insertUserData`()

创建一个过程函数，函数名为 insertUserData ，通过root 用户进入localhost连接。该语句中的root 、localhost、insertUserData 均可以不带 ``。

2. 开始结束标识

BEGIN
...
END

BEGIN和END分别为函数的开始标识和结束标识。过程函数中的实现要写在 BEGIN和END之间。

3. 变量定义

整数定义如下：

DECLARE i INT DEFAULT 1;

该语句定义了一个整数变量，名为 i，类型为 INT ，默认值为 1。

字符串定义如下：

DECLARE name VARCHAR(40) DEFAULT 'zhangsan';

该语句定义了一个字符串变量，名为 name，类型为 VARCHAR ，长度为 40，默认值为 zhangsan。

4. 事务

start TRANSACTION;
...
commit;

start TRANSACTION;和commit;分别为事务的开始标识和提交标识。在这两行语句之间的代码，在执行时会自动启动事务，也就是要么全部执行成功，要么全部失败。

5. 循环

WHILE i <= 10 DO
	INSERT INTO user_info(`id`, `user_id`, `name`, `age`, `create_time`) VALUES (i, CONCAT(410, LPAD(i, 8, '0')), name, age, create_time);
	SET i = i + 1;
END WHILE;

这一部分，是实现功能的代码。循环10次，每次插入一条数据，插入数据后，循环索引 i 自增1。

6. 字符串固定长度

LPAD(i, 8, '0')

实现字符串的前补零。把 i 作为一个字符串，字符串的长度固定为8，长度不够8时，在字符串前面补 0 。

  • 当 i = 1时，LPAD(i, 8, '0') = 00000001 。

  • 当 i = 10时，LPAD(i, 8, '0') = 00000010 。

在 MySQL 中，可以使用 LPAD 函数来实现字符串的前补零。LPAD 函数的语法如下：

LPAD(string, length, pad_string)
  • string 是要处理的原始字符串。
  • length 是目标字符串的总长度。
  • pad_string 是用于填充的字符串。

7. 字符串拼接

CONCAT(410, LPAD(i, 8, '0'))

实现字符串的拼接。

  • 当 i = 1时，LPAD(i, 8, '0') = 00000001 ，CONCAT(410, LPAD) = 41000000001 。
  • 当 i = 10时，LPAD(i, 8, '0') = 00000010 ，CONCAT(410, LPAD) = 41000000010 。

在 MySQL 中，可以使用 CONCAT 函数来实现字符串的拼接。CONCAT 函数的语法如下：

CONCAT(str1, str2, ...)
  • str1 是第一个字符串。
  • str2 是第二个字符串。
